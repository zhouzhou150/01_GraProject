# 深度学习语音识别性能测度系统整体设计文档

## 1. 课题背景

随着智能语音交互在办公自动化、智能终端和在线服务中的普及，语音识别系统已经从“能否识别”逐步转向“是否识别得准、是否响应够快、用户是否愿意持续使用”。本系统围绕中文语料库下的多模型统一评测展开，重点比较 `CNN`、`RNN`、`faster-whisper`、`PaddleSpeech` 等模型在准确性、实时性、资源占用、鲁棒性和用户满意度方面的表现。

## 2. 需求分析

### 2.1 功能需求

1. 支持语音样本与标注文本导入，并生成统一格式的数据清单。
2. 支持多种语音识别模型的统一调用和切换。
3. 支持单文件评测与批量评测。
4. 支持计算准确率、实时性、资源占用、鲁棒性与用户满意度指标。
5. 支持结果持久化存储、历史记录查看、图表展示与报告导出。
6. 提供本地 Web 端和桌面端两种使用形态。

### 2.2 非功能需求

1. 系统结构清晰，便于本科论文描述和模块化开发。
2. 允许模型依赖缺失时以模拟模式运行，保证演示可用性。
3. 数据和结果采用本地文件与 `SQLite` 存储，不依赖复杂服务器部署。
4. 中文界面简洁直观，适合毕业答辩演示。

## 3. 可行性分析

- Python 生态成熟，便于调用音频处理、深度学习、可视化和数据库组件。
- `Streamlit` 可快速构建实验型界面，`PySide6` 可对其做桌面封装。
- `SQLite` 轻量易用，适合本科毕设规模的数据管理。
- 系统支持“真实模型 + 模拟模型”双模式，便于先完成闭环，再逐步接入真实后端。

## 4. 评测指标

### 4.1 基础识别指标

- `CER`
- `WER`
- `SER`
- `SemDist Score`

### 4.2 实时性指标

- 平均延迟
- `P95` 延迟
- `RTF`
- 吞吐量
- `UPL`

### 4.3 资源占用指标

- `CPU` 使用率
- 内存占用
- `GPU` 显存占用
- 模型加载时间

### 4.4 鲁棒性指标

- 安静场景性能
- 轻噪声场景性能
- 中噪声场景性能
- 噪声场景性能保持率

## 5. 用户满意度设计

系统引入综合用户满意度指标 `USS`。该指标采用“文献权重 + 问卷校准”的混合方案：

- 准确性与语义正确性：`0.45`
- 用户感知时延：`0.25`
- 鲁棒性与稳定性：`0.20`
- 资源友好度：`0.10`

若导入问卷结果，则按 `W_final = 0.7 * W_lit + 0.3 * W_survey` 融合权重。

```text
AccuracySemanticScore = 0.6 * CER_score + 0.4 * SemDist_score
LatencyScore = 0.7 * UPL_score + 0.3 * RTF_score
RobustnessScore = 噪声场景性能保持率
ResourceScore = CPU、内存、显存、加载时间逆向归一化综合分
USS = Σ(W_final_i * Score_i)
```

满意度等级划分：

- `USS >= 85`：高满意
- `70 <= USS < 85`：良好
- `60 <= USS < 70`：一般
- `USS < 60`：待优化

## 6. 总体架构

系统采用单体分层架构：

1. 界面层：`Streamlit Web + PySide6`
2. 控制层：实验调度与任务执行
3. 模型适配层：统一封装四类模型
4. 指标计算层：文本指标、性能指标、满意度指标
5. 存储层：`SQLite + JSON/CSV/Markdown`
6. 数据层：数据集清单、音频文件、实验结果

## 7. 模块设计

### 7.1 数据管理模块

- 读取数据清单
- 检查音频路径和标注内容
- 读取样本时长、场景标签、口音标签

### 7.2 模型管理模块

- 提供统一接口 `ModelAdapter`
- 支持真实模型和模拟模型两种运行方式
- 提供模型元数据查询、热身、加载与卸载

### 7.3 指标计算模块

- 负责 `CER`、`WER`、`SER`、`SemDist`
- 负责延迟、`RTF`、吞吐量与资源监控
- 负责 `USS` 综合评分

### 7.4 任务执行模块

- 支持单文件和批量实验
- 负责时间统计、异常处理与结果聚合

### 7.5 报告分析模块

- 生成汇总表
- 输出 Markdown 报告
- 输出 JSON 与 CSV 结果

## 8. 数据流与流程

1. 用户导入数据集清单
2. 用户选择模型和测试集
3. 系统加载模型并执行评测
4. 系统采集识别结果、时延、资源占用
5. 系统计算基础指标和 `USS`
6. 结果入库并在前端展示
7. 用户导出实验报告

## 9. 数据库设计

系统采用 `SQLite`，主要表如下：

1. `experiments`
2. `sample_results`
3. `aggregate_results`
4. `export_records`

## 10. 界面设计

### 10.1 Web 端页面

- 首页
- 数据集管理
- 模型管理
- 测试任务
- 结果分析

### 10.2 桌面端

- 顶部工具栏：启动、刷新、停止
- 中部区域：嵌入 Web 页面
- 底部状态栏：服务地址、运行状态

## 11. 测试方案

### 11.1 单元测试

- 文本指标计算
- 满意度计算
- 数据库存储
- 结果导出

### 11.2 集成测试

- 单模型批量评测
- 多模型统一评测
- 数据入库与历史查询
- Web 与桌面端协同启动

### 11.3 开发过程要求

每完成一个模块后，必须立即补充对应测试并执行，确认通过后再继续下一个模块开发。

## 12. 部署方案

1. 推荐创建 Python 3.11 虚拟环境
2. 安装核心依赖与可选模型依赖
3. 生成示例数据集或导入真实数据集
4. 启动 Streamlit Web 应用
5. 如需桌面端，使用 PySide6 启动器打开本地页面
